Optional Readonly healthCheckTimeout

healthCheckTimeout?: number
  • Defined in src/core/sdk/models/configuration.ts:69

Before the mainRpc link is applied to the sdk, all the mainRpc links will be health-checked by receiving and verifying the predefined data. If an error occurs during the request the mainRpc will be replaced with a spare one. This healthCheckTimeout parameter allows you to set the maximum allowable timeout when checking the mainRpc.
